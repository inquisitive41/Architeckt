# Руководство контрибьютора Architeckt

Прежде всего, спасибо, что решили внести свой вклад в развитие Architeckt! Именно благодаря сообществу проект становится лучше.

## 1. С чего начать?

Если вы нашли баг или у вас есть идея для новой функции, проверьте раздел [Issues](../../issues) — возможно, кто-то уже создал аналогичный тикет. Если нет, смело открывайте новый!

## 2. Сделайте форк и создайте ветку

Сделайте форк репозитория Architeckt и создайте новую ветку с понятным названием (например, `feature/adasnorm` или `bugfix/attention-nan`).

## 3. Запустите тесты

Убедитесь, что ваше окружение настроено согласно [INSTALLATION.ru.md](docs/INSTALLATION.ru.md).
Запустите тесты, чтобы убедиться, что всё работает:

```bash
export PYTHONPATH="src"
python -m pytest tests/
```

## 4. Напишите код

Теперь вы готовы писать код. Не стесняйтесь просить помощи — мы открыты к диалогу!

> [!IMPORTANT]
> - Обязательно добавляйте аннотации типов (type hints) к новым функциям.
> - Соблюдайте стандарты форматирования (мы используем `black` и `flake8`).

## 5. Создайте Pull Request

Перед созданием PR убедитесь, что ваша ветка `master` синхронизирована с оригинальным репозиторием:

```bash
git remote add upstream git@github.com:your-org/Architeckt.git
git checkout master
git pull upstream master
```

Затем обновите вашу ветку, запушьте изменения и откройте Pull Request на GitHub. Мы постараемся проверить его как можно скорее!
