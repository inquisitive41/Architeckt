> **Внимание:** Эмпирически подтверждена возможность стабильного обучения на моделях малого масштаба. Масштабирование до миллиардов параметров требует отдельной экспериментальной проверки.

Architeckt — это экспериментальная архитектура больших языковых моделей (LLM) нового поколения. Она создана, чтобы преодолеть фундаментальные ограничения классических Трансформеров, предлагая **линейную вычислительную сложность O(L)**, экстремальную разреженность вычислений и адаптивную эффективность.

С Architeckt вы получаете энциклопедические знания тяжелой 11-миллиардной модели, скорость легковесной 1.5-миллиардной модели и потребление памяти, позволяющее анализировать бесконечно длинные тексты без ошибок Out-Of-Memory (OOM).

## 🌟 Ключевые возможности (Девять инноваций)

Architeckt объединяет передовые исследования и математические принципы:

* **Multi-Scale Linearized Attention (MSLA)**: Механизм внимания с линейной сложностью $O(L)$. Квадратичный барьер снят. Бесконечный контекст стал реальностью.
* **Sparse Mixture of Experts (SMoE)**: 11 миллиардов параметров, но для каждого токена активируются только ~1.5 миллиарда. Мощь тяжёлой модели при скорости лёгкой.
* **Adaptive Head Gating (AHG)**: Динамическое отключение голов внимания на простых токенах. До 30% экономии вычислений без потери качества.
* **SwiGLU-T**: Функция активации с обучаемым порогом. Создаёт естественную разреженность: не все нейроны активны одновременно.
* **AdaSNorm**: Адаптивная нормализация, заменяющая фиксированный LayerNorm. Обеспечивает стабильность глубоких сетей без ручной настройки.
* **Depth-Aware Early Exit (DAEE)**: Ранний выход из глубоких слоёв при достижении высокой уверенности. Простые токены обрабатываются быстрее.
* **Token-Level Confidence Gating (TLCG)**: Проверка уверенности модели перед выдачей каждого токена. Математическая, а не эвристическая защита от галлюцинаций.
* **Weight Tying**: Входной эмбеддинг совмещён с выходным слоем. Меньше параметров, лучше регуляризация.
* **DeepNet Initialization**: Встроенное масштабирование остаточных связей для стабильного обучения глубоких архитектур.

## 💼 Бизнес-ценность и применение

Где традиционные трансформеры упираются в ограничения, Architeckt открывает новые рынки:

* **Длинные документы:** Юридический анализ, научные публикации, техническая документация. Линейная сложность позволяет обрабатывать тексты произвольной длины без потери качества.
* **Реальное время:** Чат-боты, голосовые ассистенты, автоматическая модерация. Early Exit и Adaptive Head Gating обеспечивают минимальную задержку.
* **Ограниченные ресурсы:** Запуск на потребительских GPU (24GB VRAM), периферийных устройствах, встроенных системах. 8-битный режим и разреженная активация делают это возможным.
* **Высокие требования к достоверности:** Медицина, финансы, юриспруденция. Confidence Gating (TLCG) блокирует выдачу неподтверждённой информации (защита от галлюцинаций).
* **Снижение стоимости R&D:** Собственная модель обучается с нуля за 7 дней на 8×A100 вместо месяцев на огромных кластерах. Время выхода на рынок сокращается радикально, а активация лишь 15% параметров пропорционально снижает затраты на инференс.

## 📁 Структура проекта

Репозиторий следует лучшим практикам MLOps:

```text
ARCHITECKT/
├── configs/            # YAML конфигурации для моделей и тренировки
├── data/               # Датасеты и бинарные токенизированные файлы
├── docs/               # Подробная документация
├── results/            # Результаты экспериментов, метрики, логи
├── scripts/            # Bash/PowerShell скрипты автоматизации
├── src/                # Ядро архитектуры (исходный код)
│   ├── activations/    # SwiGLU-T и кастомные активации
│   ├── attention/      # MSLA, Adaptive Head Gating, RoPE
│   ├── benchmarks/     # Теоретический расчет FLOPs, памяти, задержек
│   ├── inference/      # Утилиты инференса (8-bit загрузка)
│   ├── layers/         # Блоки трансформера
│   ├── models/         # Определение классов модели
│   ├── normalization/  # Слой нормализации AdaSNorm
│   ├── routing/        # Роутер токенов (CATR)
│   └── training/       # Обучающий цикл (FSDP Trainer)
└── tests/              # Юнит-тесты (Pytest)
```

## 🚀 Быстрый старт

> [!IMPORTANT]
> Для работы Architeckt требуется GPU с поддержкой CUDA. Для обучения с нуля рекомендуется кластер 8×A100. Для инференса достаточно одной видеокарты на 24GB VRAM.

### 1. Установка

Подробные инструкции по настройке окружения доступны в [Руководстве по установке](docs/INSTALLATION.ru.md).

```bash
git clone https://github.com/your-org/Architeckt.git
cd Architeckt
pip install -r requirements.txt
```

### 2. Верификация возможностей архитектуры

Запустите скрипты бенчмарков, чтобы убедиться в колоссальной экономии памяти и вычислительных ресурсов:

```bash
# Настройка переменной окружения
export PYTHONPATH="src"

# Расчет теоретической задержки и памяти
python src/benchmarks/latency.py
python src/benchmarks/memory.py
```

### 3. Пример Инференса (8-bit)

> [!NOTE]
> Убедитесь, что установлена библиотека `bitsandbytes` для загрузки 11B модели в 24GB VRAM.

```python
import torch
from src.inference.inference_utils import load_for_inference
from src.models.architeckt import ArchitecktModel  # Скоро в релизе!

# Инициализация каркаса модели
model = ArchitecktModel(config)

# Загрузка 8-битных весов для ограниченной VRAM
model = load_for_inference(model, "checkpoints/architeckt_final.pt", use_8bit=True)

# Генерация
# outputs = model.generate(...)
```
