# Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Чем Architeckt отличается от LLaMA или GPT?
Традиционные модели используют Softmax-внимание, которое масштабируется квадратично ($O(L^2)$) от длины текста. Architeckt использует **Multi-Scale Linearized Attention (MSLA)**, масштабирующееся линейно ($O(L)$). Это означает, что Architeckt может читать бесконечно длинные тексты без падения производительности. Кроме того, модель использует Sparse Mixture of Experts (SMoE) и адаптивное отключение голов (AHG), чтобы радикально снизить количество вычислений на каждое слово.

### Какой лимит контекста у Architeckt?
Теоретически, **жесткого лимита контекста нет**. KV-кэш не растет с увеличением длины текста. На практике модель ограничена лишь обученными позиционными эмбеддингами (RoPE), но их можно легко экстраполировать (расширять) с помощью методов вроде YaRN или Position Interpolation.

## Решение проблем (Troubleshooting)

### "CUDA Out Of Memory" при Инференсе
**Проблема:** Возникает ошибка OOM при попытке загрузить модель на видеокарту 24GB (RTX 3090/4090).
**Решение:** Полная 11-миллиардная модель весит ~23 ГБ в формате `bf16`. Чтобы она стабильно работала на 24GB, необходимо 8-битное квантование. Убедитесь, что установлена библиотека `bitsandbytes`, и передайте флаг `use_8bit=True` в функцию `load_for_inference`.

### Ошибка "ModuleNotFoundError: No module named 'bitsandbytes'" на Windows
**Проблема:** Библиотека не устанавливается или не импортируется на ОС Windows.
**Решение:** `bitsandbytes` официально поддерживает только Linux. На Windows у вас есть два варианта:
1. Запустить проект через **WSL2** (Windows Subsystem for Linux).
2. Установить неофициальную сборку для Windows: `python -m pip install bitsandbytes-windows`.

### Скачки Loss / NaN во время обучения
**Проблема:** Значение Loss (потерь) внезапно взлетает до бесконечности (NaN) при смешанной точности.
**Решение:** 
- Убедитесь, что вы используете `bfloat16`, а не `float16`. Стандартному `fp16` не хватает динамического диапазона (экспоненты) для корректного накопления кумулятивных сумм в Линейном Внимании.
- Убедитесь, что включено клиппирование градиентов (`max_grad_norm=1.0`).

### Ошибка FSDP "ProcessGroupNCCL"
**Проблема:** `ArchitecktTrainer` падает с ошибкой NCCL при инициализации.
**Решение:** Обычно это происходит, если вы запускаете код на Windows без нужных бэкендов, либо на одной видеокарте без настройки мультипроцессинга. PyTorch FSDP требует полноценного Linux-окружения с бэкендом `nccl`. Если вы просто тестируете код на одном GPU локально, установите `use_fsdp=False` в `TrainingConfig`.
